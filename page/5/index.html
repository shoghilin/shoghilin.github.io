資料中常見的特徵類型

發表於 2020-05-23 更新於 2020-05-24 分類於 程式筆記

常見兩大類型(數值型 numerical data、類別型 categorical data )

數值型特徵 : 有不同轉換方式，函數 / 條件式都可以

最容易轉成特徵，但需要注意很多細節

  • e.g.(函數)
    數值： 20坪
    函數： 一坪20萬
  • e.g.(條件式)
    年齡： 30
    條件： 若 50以上：0點，30～50：1點，30以下：2點

類別型特徵 : 通常一種類別對應一種分數

通常一種類別對應一種分數，問題在如何對應

其他類型

二元特徵

僅有True(1)跟False(0)兩特徵
可作為數值、亦可作為類別

排序型特徵

如名次/百分等級，有大小關係，但並非連續數字
通常做為數值型，因做為類別型會失去排序資料

時間型特徵

e.g. 2020/05/23 20:49:00
有週期性
雖然時間型特徵可當作數值型特徵或類別型特徵，但都不適合

資料類型及類別轉數值方式

發表於 2020-05-22 更新於 2020-06-11 分類於 程式筆記

在初步 EDA 的過程，我們無可避免會想問的問題：

不同資料類型各有多少個欄位？

類別型欄位 (pandas 中的 object) 的類別數量?

模型怎麼處理類別型的資料？有什麼表示方法？

欄位變數可分為:

  1. 離散變數：只能用整數單位計算的變數
    如：性別(僅有男、女、雙性的選擇)
  2. 連續變數：在一定區間可任意取值的變數
    如：身高(可能在0~230，任意數)

DataFrame中常見資料類型分為：

  1. int64 ： 離散、連續變數
  2. float64 ： 離散、連續變數
  3. object ： 類別型變數

字串/類別類型的資料，若需要轉換為數值的資料類型。

常見轉換有兩種:

Label encoding：

使用時機通常是該資料的不同類別是有序的
把每個類別 mapping 到某個整數，不會增加新欄位
如：年紀(老、中、小)有序，則設老1、中2、小3之類的
空間及計算時間較小，適合非深度學習模型

One Hot encoding：

使用時機通常是該資料的不同類別是無序的
為每個類別新增一個欄位，用 0/1 表示是否
如：國家，則設美國[1 0 0]’，日本[0 1 0]’，台灣[0 0 1]’之類
空間及計算時間較大，適合深度學習模型
因此當特徵重要性高，且可能值較少時，才應該考慮獨熱編碼

綜合

非深度學習時，類別型特徵建議預設採Label encoding;
深度學習時，預設採One Hot encoding.

因非深度學習時主要是樹狀模型 (隨機森林 / 梯度提升樹等基於決策樹的模型），用兩次門檻就能分隔關鍵類別;
但深度學習主要依賴倒傳遞，Label encoding會不易收斂

實作方式

這兩種轉換法被包括在SciKit Learn library中
ps. Label encoding被ColumnTransformer給替代了

Label encoder

把每個類別 mapping 到某個整數，不會增加新欄位

存在的問題：被轉換的column並不具有關聯性，然而我們的model卻可能誤認為其中具有排序關係

One Hot Encoder

若類別中無排序關係，則應使用OneHotEncoder
先將類別做Label Encoding，然後為各類別新增一個欄位，並以0s或1s表示

為何建立需要dataframe

發表於 2020-05-21 更新於 2020-05-22 分類於 程式筆記

為甚麼需要新建dataframe

  1. 需要將分析過程中產生的數據或結果儲存為結構性資料
  2. 當資料過於複雜時，可簡化資料做程式測試

不同的資料有不同讀取⽅方式

  • 文字格式通常可以⽤ with open()
  • 圖像格式可以使⽤用 PIL(image), skimage 或是 cv2
    CV2 的速度較快，但須注意讀入的格式為 BGR
  • 其他形式如 npy / pickle 可以儲存經過處理理後的資料

分位數 quantile (document)

將資料分成數等分，其分割點即稱為分位數

分位數例子：

收集11位同學在罰球線上投籃10次進籃的次數，每位同學
投中的次數分別為
3 2 3 7 4 3 6 4 3 3 6
何者為投中次數的第一與第三四分位數?

解說：

先將11位同學投中的次數由小到大排序，為
2 3 3 3 3 3 4 4 6 6 7
因為資料個數(11位同學)為奇數，最中間的位置為第6位，所以中位數為3。
小於中位數的數值為第1至第5筆資料，為2 3 3 3 3，
所以此部份的中位數為第3筆資料、數值為3，代表第一四分位數為6。
大於中位數的數值為第7至第11筆資料，為4 4 6 6 7，
所以此部份的中位數為第9筆資料、數值為6，代表第三四分位數為6。
