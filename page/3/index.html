Kernel Density Estimation (KDE)

什麼是KDE

KDE是採⽤無⺟數⽅法畫出⼀個觀察變數的機率密度函數，其中kernel在“非參數估計”的語境下，“kernel”是一個函數，用來提供權重。例如高斯函數 (Gaussian) 就是一個常用的kernel function。

Density plot 的特性

  • 歸⼀：線下 ⾯積和為 1
  • 對稱：K(-u) = K(u)

常⽤的 Kernel function

  • Gaussian (Normal dist)
  • Cosine

優缺點

優點：

  • 無母數方法，分布沒有假設

缺點：

  • 計算量大

相關係數

何謂相關係數

相關係數是其中一個常用來了解各個欄位與我們想要預測的目標之間的關係的指標。
公式為

$ r = \frac {1}{n-1}\sum_{i=1}^n {(\frac{x_i - x'}{s_x})}{(\frac{y_i - y'}{s_y})}$

如果我們想要了解兩個變數之間的線性關係時，相關係數就會是一個還不錯的簡單方法 他會產生介於 -1～1 之間的值，負值代表負相關，正值代表正相關，而數值的大小代表相關性的強度，通常會搭配散佈圖scatter 如： * 00-.19: 非常弱相關 * .20-.39: 弱相關 * .40-.59: 中度相關 * .60-.79: 強相關 * .80-1.0: 非常強相關

python中使用

np.corrcoef(x, y)
>>
[x跟x的相關係數, x跟y的相關係數]
[y跟x的相關係數, y跟y的相關係數]

outliers 或 NA 填補

補統計值

  • 平均數 (mean)
    數值型欄位，偏態不明顯

    np.mean(value_array)

  • 中位數 (median, or Q50)
    數值型欄位，偏態很明顯

    np.median(value_array)

    or

    numpy.percentile(value_array, q=50)

  • 眾數
    類別型欄位

    from scipy.stats import mode
scipy.stats.mode(value_array): 較慢的方法(未辨明)

    or

    dictionary method: 較快的方法(未辨明)

    or

    list(app_train['AMT_GOODS_PRICE'].value_counts().index)[0]
# 取眾數最大值[0]

  • 分位數 (quantile)

    np.quantile(value_arrary, q = …)

    or

    numpy.percentile(value_array, q=[...])

填補指定值 - 需具備欄位相關領域知識

  • 補 0 : 空缺原本就有 0 的含意
  • 補不可能出現的數值 : 類別型欄位，但不適合⽤眾數時

歸一化和標準化

  • 歸一化:
    就是將訓練集中某一列數值特徵（假設是第i行）的值縮放到0和1之間[0 ~ 1]。
    最⼩最⼤化 (MinMax Scaler) :假定數值為均勻分佈，且資料不會有極端值、或已去極端值，適合本⽅式平衡特徵
    Rescaling (min-max normalization)：$ x’ = \frac{x - min(x)}{max(x) - min(x)} $
    Mean normalization：$ x’ = {\frac{x-average(x)}{max(x)-min(x)}} $

    from sklearn.preprocessing import MinMaxScaler
MinMaxScaler()
  • 標準化:
    就是將訓練集中某一列數值特徵（假設是第i行）的值縮放成均值為0，方差為1的狀態[-1 ~ 1]。
    假定數值為常態分佈，適合本⽅式平衡特徵
    Standardization：$ x’ = \frac{x - \bar{x}}{\sigma}$

好處

  • 提升模型的收斂速度
  • 提高模型精度，這在涉及到一些距離計算的算法時效果顯著

連續型數值標準化

是否一定要做標準化 (有沒有做有差嗎)

標準化的意義：平衡數值特徵間的影響⼒

看使用的模型而定
• Regression model(如線性迴歸, 羅吉斯迴歸, ANN…等)：有差
• Tree-based model(如決策樹, 隨機森林, 梯度提升樹…等)：沒有太大關係

連續值標準化

  1. Z-transform: $ \frac{(x - mean(x))}{std(x)} $
  2. Range (0 ~ 1): $ \frac{x - min(x)}{max(x) - min(x)} $
  3. Range (-1 ~ 1): $ (\frac{x - min(x)}{max(x) - min(x)} - 0.5) * 2 $

標準化的好壞

Good

  • 某些演算法 (如 SVM, DL) 等，對權重敏感或對損失函數平滑程度有幫助者
  • 特徵間的量級差異甚大

Bad

  • 有些指標，如相關不適合在有標準化的空間進行
  • 量的單位在某些特徵上是有意義的

ECDF (experience empirical distribution function)

ECDF 最主要的目的是希望看到資料的累積分布，y 軸是介於 0-1 的數字 (代表累積比例)，x 軸則是資料的實際數值分布。
透過 ECDF，我們可以很快看出我們有興趣的資料數值約占多少比例，並觀察圖形差異
而CDF跟ECDF的差別就是，CDF 是從理論函數來的，ECDF 則是由觀察值所得到的，因此CDF看起來較為平滑，ECDF則較多曲折。

離群值(Outlier)及其處理

何謂離群值(Outlier)

離群值是與正常數值偏離較遠的數值群，如果不處理則特徵縮放(標準化 / 最小最大化)就會出現很大的問題

檢查異異常值的⽅方法

  • 統計值：如平均數、標準差、中位數、分位數 (dataframe.describe)
  • 畫圖：如直方圖(hist)、盒圖(boxplot)、次數累積分布等(ECDF)

延伸資料：

Ways to Detect and Remove the Outliers
https://towardsdatascience.com/ways-to-detect-and-remove-the-outliers-404d16608dba

處理理異異常值

  • 取代補值：中位數、平均數等
  • 另建欄位
  • 整欄不用
  • 對數轉換
    對存在離群值的變量作對數轉換可以克服其離群值問題，且對數轉換並不影響各觀察值之間在此變量上的相對大小

    10 與 1000 相差百倍，但二者的對數值卻僅相差 3 倍，由此可知資料在經過對數轉換後變得更加集中
  • 縮尾
    將超出變量特定百分位範圍的數值替換為其特定百分位數值的方法。
  • 截尾
    將超出變量特定百分位範圍的數值予以刪除的方法。
  • 插值

處理完後的好處/壞處

好處是剩餘資料中模型較為單純且準確，壞處是有可能刪除掉重要資訊

高斯分佈(Gaussian distribution)

又稱為常態分布(normal distribution)，描述連續型資料呈現對稱型分布，具鐘型特性

常態分佈完美由 $\mu$ 和 $\sigma$ 所代表，並且以平均為中心做對稱分佈
其中$\mu$代表平均或分布的中心，$\sigma$代表標準差

統計量化的方式

以單變量分析來來說，量化的分析方式可包含

計算集中趨勢

  • 平均值 Mean
  • 中位數 Median
    一組資料的中位數是指將資料從小到大排序後，最中間的數
  • 眾數 Mode
    眾數指一組數據中出現次數最多的變數值

計算資料分散程度

  • 最小值 Min
  • 最大值 Max
  • 範圍 Range
  • 四分位差 Quartiles
  • 變異數 Variance
  • 標準差 $\sigma$ Standard deviation

資料平滑處理

資料平滑處理 —- log1p()和exmp1()

log1p() document
exmp1() document

  1. 資料預處理時首先可以對偏度比較大的資料用log1p函式進行轉化，使其更加服從高斯分佈，此步處理可能會使我們後續的分類結果得到一個好的結果。
  2. 平滑問題很容易處理掉，導致模型的結果達不到一定的標準，log1p( )能夠避免復值得問題 — 復值指一個自變數對應多個因變數

log1p( ) 的使用就像是一個數據壓縮到了一個區間，與資料的標準類似。其逆運算就是expm1的函式

由於使用的log1p（）對資料進行了壓縮，最後需要將預測出的平滑資料進行一個還原，而還原過程就是log1p的逆運算expm1.

當x較大時直接計算，當x較小時用泰勒展開式計算
