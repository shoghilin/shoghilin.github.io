糖尿病網膜症探知

發表於 2023-01-26

こんにちは、林です。

これから早稲田大学IPSに入学する時、研究テーマとしては糖尿病網膜症探知にする予定なので、少し自分なりに研究しました、その結果はこちらです。

背景と目的

糖尿病網膜症（とうにょうびょうもうまくしょう、英: Diabetic Retinopathy）とは、糖尿病が原因で目の中の網膜という組織が障害を受け、視力が低下する病気です。その発症率は糖尿病患者の約40％を占める。

糖尿病網膜症は進行性の疾患で、病変部位の有無で5段階に分けられ、血管新生の有無で増殖性DR（PDR）か非増殖性DR（NPDR）かが決まります。もし病気の後期段階で発見したら、部分的な視力を回復するために手術が必要になります。早期発見と早期治療により、手術を回避することできます。

糖尿病網膜症5段階

この研究では糖尿病網膜症の有無や、病気の段階を自動的に判断するシステムを作ります。

分類問題としてのアプローチ

本タスクへの分類問題に対して私のアプローチについて概要を記録します。

網膜画像が入力として与えられた際に、その画像が糖尿病網膜症のどの段階にいるのかを深層学習モデルに分類させたいと考えています。

具体的に言うと、網膜画像が深層学習モデルに入力した後、糖尿病網膜症の全段階それぞれの確率を予測して、最終的に確率が一番高いのが分類する結果となります。

本編で使われている深層学習モデルはInceptionv3[1]と言うCNNに基づくモデルで、その後で全結合層を利用して確率表示に変わります。このモデルは計算効率性がいいや、multi-scaleの構造で表現力がありなどのメリットあります。その構造は以下となります。

結果

  • 分類タスクは特に前処理しなくてもある程度の性能は発揮できるが、不均衡データ問題が存在する場合、効果は酷く影響されます。
  • 糖尿病網膜症の第一段階は他の段階と間違えやすい。
  • 不均衡データ問題に対し、本編ではweighted CrossEntropyや、focal lossを利用すると、効果は大幅上げれますが、まだまだ改善が必要です。

実験

本実験で使われた評価関数は正解率（Accuracy）、カッパ係数（Cohen’s Kappa）、F値（F-Score）、再現率（Recall）、適合率（Precision）。以下は各評価関数の式です。

$$
Accuracy=\frac{TP+TN}{TP+TN+FP+FN}
$$

$$
Recall=\frac{TP}{TP+FN}
$$

$$
Precision=\frac{TP}{TP+FP}
$$

$$
\text{F-score}=\frac{2*Precision*Recall}{Precision+Recall}
$$

その中一番特別なのはカッパ係数です。カッパ係数とは、カテゴリカルデータに対して2つの検査結果の一致度合いを評価する指標のことです。数値が高いほど、我々のモデル判断と医者の判断が一致すると意味します。式は以下となります。

$$
K=\frac{p_o-p_e}{1-p_e}
$$

その中

  • $p_o$は観察によって得られた結果確率で、ここでは医師の分類結果を意味します。
  • $p_e$はモデルからの分類結果確率。

実験では、APTOS資料集[2]とEyePACS資料集[3]、二つの資料集が使われております。

不均衡データ学習問題 - Focal Loss

この不均衡データ学習問題に対して、ここではweighted CrossEntropyだけではなく、Focal loss[4]も利用してみました。

Focal lossとは、不均衡データ学習問題に対しての損失関数で、この中心理論は簡単なものに対して小さな損失与え、難しいものに対し大きな損失与えることです。その式はこちら。

$$
L(y,\mathbf{\hat{p}})=-(1-\hat{p}_y)^\gamma \log{(\hat{p}_y)}
$$

その中で

  • $y\in {0,\dots,K-1}$は整数のクラスラベル (K はクラスの数を表す)、
  • $\mathbf{\hat{p}}=(\hat{p}_0,\dots,\hat{p}_{K-1})\in [0,1]^K$はK クラスの推定確率分布を表すベクトルです。
  • $\gamma$は信頼性の高い正しい予測が全体の損失にどの程度寄与するかを指定するフォーカス パラメータです。

これまでのまとめ[総評]

  • 網膜画像に対し何が特別な前処理しなくてもある程度の分類性能は発揮できる。
  • 不均衡データ学習問題を解決しないと理想な効果たどり着けない。
  • 正確率高いでも効果いいと言う保証はない、多方面で考える必要がある。
  • Focal lossとweighted CrossEntropyは確かに効果的ですが、まだ改善が必要です。

これからの課題

  • 一般的な分類問題と処理すると、不均衡データ学習問題が存在するため、今度は距離問題として処理したら、不均衡データを心配することないはず。
  • 前研究したテーマ、person re-identificationでは、Domain shiftと言う問題が存在しています。即ちA domainで訓練したモデルがB domainで利用し、評価されたら、結果はすごく下がる。この問題は糖尿病網膜症探知の中でも存在するかどうか試したいです。
  • 不均衡データ学習問題に対して、本編ではFocal lossとweighted CrossEntropyを利用しましたが、それ以外の解決方法（例えばover-samplingや、under-samplingとか）も試したいです。
  • 網膜画像はある程度の前処理でもっと区別性を強化できると思います。

池化(Pooling)層與參數調整

發表於 2020-08-18 分類於 程式筆記

池化(Pooling)層與參數調整

常用參數：

  • pool_size：整數，沿（垂直，水平）方向縮小比例的因數
    (2，2)會把輸入張量的兩個維度都縮小一半
  • strides：整數，2 個整數表示的元組，或者是”None”。表示步長值。
    如果是 None，那麼默認值是 pool_size
  • padding：”valid”或者”same”（區分大小寫)
  • data_format：channels_last(默認)或 channels_first 之一。表示輸入各維度的順序
    channels_last 代表尺寸是(batch, height, width, channels)的輸入張量
    channels_first 代表尺寸是(batch, channels, height, width)的輸入張量

池化層(Pooling Layer) 超參數

前端輸入feature map 維度：W1×H1×D1
有兩個hyperparameters：

  • Pooling filter 的維度- F,
  • 移動的步數 S,

所以預計生成的輸出是 W2×H2×D2:
W2=(W1−F)/S+1W2=(W1−F)/S+1
H2=(H1−F)/S+1H2=(H1−F)/S+1
D2=D1

Pooling Layer (池化層) 適用的場景

特徵提取的誤差主要來自兩個方面

  1. 鄰域大小受限造成的估計值方差增大
  2. 卷積層超參數與內核造成估計均值的偏移

一般來說：

  • average-pooling 能減小第一種誤差，更多的保留圖像的背景信息
  • max-pooling 能減小第二種誤差，更多的保留紋理信息

卷積(Convolution)層與參數調整

卷積 (Convolution) 的 超參數(Hyper parameter )

常用超參數

  • filters: 濾波器的數量。此數字會等於做完卷積後特徵圖的通道數，通常設定為 2 的 n 次方
  • kernel_size: 濾波器的大小。通常都是使用 3x3 或是 5x5
  • input_shape: 只有對影像做第一次卷積時要指定，之後 Keras 會自動計算 input_shape
  • strides: 做卷積時，濾波器移動的步長。
  • padding: 是否要對輸入影像的邊緣補值。

填充或移動步數(Padding/Stride )的用途

padding = ‘VALID’ 代表不用填補。也就是等於最一開始敘述的卷積計算，圖根據 filter 大小和 stride大小而變小

$$new_{height} = new_{width} = (W - F + 1) / S$$

也就是
10x10的資料
filter 3x3, stride=1, 卷積後的大小: (10–3+1)/1=8
filter 3x3, stride=2, 卷積後的大小: (10–3+1)/2=4

padding = ‘Same’ 表示使用填補方式使輸出和輸入具有相同的寬度和高度。

pad=1，表示圖外圈額外加 1 圈 0，假設 pad=2，圖外圈額外加 2 圈 0，以此類推
(1) kernel size是 3 的時候，卷積後圖的寬高不要變，pad 就會設定為 1
(2) kernel size是 5 的時候，卷積後圖的寬高不要變，pad 就會設定為 2

卷積神經網路(CNN)特性

適合用在影像上

  • 因為 fully-connected networking 如果用在影像辨識上，會導致參數過多(因為像素很多)，導致 over-fitting
  • CNN 針對影像辨識的特性，特別設計過，來減少參數
  • Convolution：學出 filter 比對原始圖片，產生出 feature map (也當成image)
  • Max Pooling：將 feature map 縮小
  • Flatten：將每個像素的 channels (有多少個 filters) 展開成 fully connected feedforward network

卷積神經網路跟深度網路

傳統的DNN（即Deep neural network）最大問題在於它會忽略資料的形狀
例如，輸入影像的資料時，該 data 通常包含了水平、垂直、color channel 等三維資訊，但傳統 DNN 的輸入處理必須是平面的、也就是須一維的資料。
一些重要的空間資料，只有在三維形狀中才能保留下來。

然而，RGB 不同的 channel 之間也可能具有某些關連性、而遠近不同的像素彼此也應具有不同的關聯性

而深度學習(Deep learning)中的 CNN 較傳統的 DNN 多了 Convolutional（卷積）及池化（Pooling) 兩層layer，用以維持形狀資訊並且避免參數大幅增加。

Convolution原理是透過一個指定尺寸的window，由上而下依序滑動取得圖像中各局部特徵作為下一層的輸入，這個sliding window在CNN中稱為稱為Convolution kernel
利用此方式來取得圖像中各局部的區域加總計算後，透過 ReLU activation function輸出為特徵值後再提供給下一層使用

卷積網路的組成

一個CNN主要是由

  1. Convolution Layer 卷積層
  2. Pooling Layer 池化層
  3. Flatten Layer 平坦層
  4. Fully connection Layer 全連接層

Pooling - 池化層(更多)

Pooling layer 稱為池化層，它的功能很單純，就是將輸入的圖片尺寸縮小（大部份為縮小一半）以減少每張 feature map 維度並保留重要的特徵，
其好處有：

  • 特徵降維，減少後續 layer 需要參數
  • 具有抗干擾的作用：圖像中某些像素在鄰近區域有微小偏移或差異時，對 Pooling layer 的輸出影響不大，結果仍是不變的
  • 減少過度擬合 over-fitting 的情況

與卷積層相同，池化層會使用kernel來取出各區域的值並運算，但最後的輸出並不透過Activate function（卷積層使用的function是ReLU）

keras 實作

Flatten - 平坦層

Flatten：將特徵資訊丟到 Full connected layer 來進行分類，其神經元只與上一層 kernel 的像素連結，而且各連結的權重在同層中是相同且共享的

keras 實作

Fully connection - 全連接層

卷積(convolution)和池化層(pooling)，其最主要的目的分別是 提取特徵及減少圖像參數 ，然後將特徵資訊丟到 Full connected layer 來進行分類，其神經元只與上一層 kernel的像素連結，而且各連結的權重在同層中是相同且共享的

影像辨識的傳統方法

影像辨識的傳統方法是特徵描述及檢測，需要辦法把影像像素量化為特徵（特徵工程），然後把特徵丟給我們之前學過的機器學習算法來做分類或回歸

傳統電腦視覺提取特徵的方法有

1. 顏色直方圖

顏色直方圖是將顏色信息轉化為特徵一種方法，將顏色值 RGB 轉為直方圖值，來描述色彩和強度的分佈情況。舉例來說，一張彩色圖有 3 個 channel， RGB，顏色值都介於 0-255 之間，最小可以去統計每個像素值出現在圖片的數量，也可以是一個區間如 (0 - 15)、(16 - 31)、…、(240 - 255)。可表示如右圖

直方圖

通過調用 cv2.calcHist(images, channels, mask, histSize, ranges) 函數來得到統計後的直方圖值

images (list of array)：要分析的圖片
channels：產生的直方圖類型。例：[0]→灰度圖，[0, 1, 2]→RGB三色。
mask：optional，若有提供則僅計算 mask 部份的直方圖。
histSize：要切分的像素強度值範圍，預設為256。每個channel皆可指定一個範圍。例如，[32,32,32] 表示RGB三個channels皆切分為32區段。
ranges：像素的範圍，預設為[0,256]，表示<256

方向梯度直方图HOG(Histogram of Oriented Gradient) (更多)

HOG特徵是一種在計算機視覺和圖像處理中用來進行物體檢測的特徵描述子。
HOG不是基於顏色值而是基於梯度來計算直方圖的，它通過計算和統計圖像局部區域的梯度方向直方圖來構建特徵。
HOG特徵結合SVM分類器已經被廣泛應用到圖像識別中，尤其在行人檢測中獲得了極大的成功。

  • 我們可以透過監控 validation loss 來保證所存下來的模型是在 validation set 表現最好的一個。
  • 假如要做 Inference :可以保證使用的是對 monitor metric 最佳的權重

keras 實作

Reduce Learning Rate

隨訓練更新次數，將 Learning rate 逐步減小

  • 因為通常損失函數越接近谷底的位置，開口越小 – 需要較小的 Learning rate 才可以再次下降

可行的調降方式

  • 每更新 n 次後，將 Learning rate 做一次調降 – schedule decay
  • 當經過幾個 epoch 後，發現 performance 沒有進步 – Reduce on plateau

優化器optimizers

  • 機器學習算法當中，大部分算法的本質就是建立優化模型，通過最優化方法對目標函數進行優化從而訓練出最好的模型
  • 優化算法的功能，是通過改善訓練方式，來最小化(或最大化)損失函數 E(x)
  • 優化策略和算法，是用來更新和計算影響模型訓練和模型輸出的網絡參數，使其逼近或達到最優值

最常用的優化算法-Gradient Descent

通過尋找最小值，控制方差，更新模型參數，最終使模型收斂

$$ w_{i+1} = w_i - d_i·\eta_i, i=0,1,…$$
參數 $\eta$ 是學習率。這個參數既可以設置為固定值，也可以用一維優化方法沿著訓練的方向逐步更新計算
參數的更新分為兩步：第一步計算梯度下降的方向，第二步計算合適的學習
