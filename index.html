均值編碼和平滑化

均值編碼和平滑化

均值編碼 (Mean Encoding)

什麼是均值編碼？

使用目標值的平均值，取代原本的類別型特徵

甚麼時候使用

如果類別特徵看起來與目標值有顯著相關，那麼均值編碼便是一個不錯的選擇

缺點

然而均值編碼的缺點是如果交易樣本非常少, 且剛好抽到極端值, 平均結果可能會有誤差很大。
如：
各籃球隊中鋒身高有{180,175,220,200,210}，然而若是抽到175，並以此作均值編碼，便會產生極大誤差

怎麼解決呢

若要解決這個缺點均值編碼還需要考慮紀錄筆數, 以當作可靠度的參考，也就是平滑化 (Smoothing)

其中一種方式如下：
$$新類別均值 = \frac{原類別平均 * 類別樣本數 + 全部的總平均 * 調整因子}{類別樣本數 + 調整因⼦子}$$
調整因子用來調整平滑化的程度，依總樣本數調整

當平均值的可靠度低時, 我們會傾向相信全部的總平均
當平均值的可靠度⾼時, 我們會傾向相信類別的平均
依照紀錄筆數, 在這兩者間取折衷

需要注意

當使用均值編碼時，非常容易造成overfitting，即便使用平滑化能夠修正，但實際效果有限。
因此仍須 經過檢驗 後再決定是否該使用均值編碼

計數編碼/雜湊編碼

計數編碼 (Counting)

什麼是記數編碼

如果類別的 目標平均值 與 類別筆數 呈正相關(或負相關)，也可以將筆數本⾝身當成特徵
例如 : 購物網站的消費金額預測

p.s. 自然語言處理時，字詞的計數編碼又稱詞頻，本身就是一個很重要的特徵

實作例子

特徵雜湊 ( Feature Hash ) wiki

當類別型特徵最麻煩的問題 : 相異異類別的數量非常龐大, 該如何編碼?
然而這個問題並沒有通用的解決方法，只能通過折衷的方式達成
也就是 特徵雜湊

什麼是特徵雜湊

特徵雜湊是一種折衷方案，將類別由雜湊函數定應到一組數字，調整雜湊函數對應值的數量，
在計算空間/時間與鑑別度間取折衷，也提高了訊息密度, 減少無用的標籤

什麼時候使用

當相異類別數量相當大且其他編碼方式效果更差時，可以考慮雜湊編碼以節省時間

特徵工程 數值型特徵 - 去除偏態

去除偏態的目標在於讓數值更接近常態分布(左右對稱，集中點在中央)，讓平均值更具有代表性。
當離群資料比例太高，或者平均值沒有代表性時，皆可以考慮去除偏態。

去偏方式

  • 對數去偏 log1p
    對數去偏就是使用自然對數(log)去除偏態，常見用於非負值且可能為0的值
    為了使對數0為0，需要在後面加上1(plus 1 -> 1p)後，再取對數
    還原時用expm1(exp + minus 1)

  • 方根去偏(sqrt)
    方根去偏就是將數值減去最⼩值後開根號，最大值有限的時候適用 (例 : 成績轉換)

  • 分布去偏(boxcox)
    分布去偏是採用boxcox轉換函數(詳情如下表)，函數的 lambda(λ) 參參數為 0 時
    等於 log 函數，lambda(λ) 為 0.5 時等於開根號 (即sqrt)，因此可藉由參數的調整更靈
    活地轉換數值，但要特別注意Y的輸入數值必須要為正 (不可為0)

特徵工程

特徵工程是事實對應到後續評估分數的轉換

特徵工程的 目的 就是最大限度地從原始數據中提取特徵以供算法和模型使用。
以鐵達尼號中的生存為例，若從性別提取特徵，為男性評估點數為-2，女性評估為1
假設有一資料組性別為男則給予他-2的點數，當然此點數本身未必會直接對應到機率，但會是後續評估的依據

通常來說特徵工程會在資料彙整後，擬和模型前。

由於資料包含類別型(文字型)特徵以及數值型特徵，
所以通常特徵工程⾄少要包含一種類別編碼(LabelEncoder/OneHotEncoder)，以及一種特徵縮放方法(標準化或歸一化)

離散化連續數值的意義以及方法

離散化的目的

  • 簡化model(減少可能性)
    原年齡有0～100歲的可能，若以十歲間距離散，就剩下10種可能
  • 變數較穩定
    若有異常值(如年齡=300)可直接設超過60為1，其他為0，降低他的影響力

關鍵點

  • 組的數量
    以年齡為例，並設10歲為間距，即可分成10組
  • 組的寬度
    一組的寬度是 10 歲

離散化的方法

主要的方法

  • 等寬劃分(pd.cut())：按照相同寬度將資料分成幾等份。缺點是受到異異常值的影響比較⼤大。
  • 等頻劃分(pd.qcut())：將資料分成幾等份，每等份資料裡面的個數是一樣的。
  • 聚類劃分：使⽤用聚類演算法將資料聚成幾類，每一個類為一個劃分。

相關係數

何謂相關係數

相關係數是其中一個常用來了解各個欄位與我們想要預測的目標之間的關係的指標。
公式為

$ r = \frac {1}{n-1}\sum_{i=1}^n {(\frac{x_i - x'}{s_x})}{(\frac{y_i - y'}{s_y})}$

如果我們想要了解兩個變數之間的線性關係時，相關係數就會是一個還不錯的簡單方法 他會產生介於 -1～1 之間的值，負值代表負相關，正值代表正相關，而數值的大小代表相關性的強度，通常會搭配散佈圖scatter 如： * 00-.19: 非常弱相關 * .20-.39: 弱相關 * .40-.59: 中度相關 * .60-.79: 強相關 * .80-1.0: 非常強相關

python中使用

np.corrcoef(x, y)
>>
[x跟x的相關係數, x跟y的相關係數]
[y跟x的相關係數, y跟y的相關係數]

outliers 或 NA 填補

補統計值

  • 平均數 (mean)
    數值型欄位，偏態不明顯

    np.mean(value_array)

  • 中位數 (median, or Q50)
    數值型欄位，偏態很明顯

    np.median(value_array)

    or

    numpy.percentile(value_array, q=50)

  • 眾數
    類別型欄位

    from scipy.stats import mode
scipy.stats.mode(value_array): 較慢的方法(未辨明)

    or

    dictionary method: 較快的方法(未辨明)

    or

    list(app_train['AMT_GOODS_PRICE'].value_counts().index)[0]
# 取眾數最大值[0]

  • 分位數 (quantile)

    np.quantile(value_arrary, q = …)

    or

    numpy.percentile(value_array, q=[...])

填補指定值 - 需具備欄位相關領域知識

  • 補 0 : 空缺原本就有 0 的含意
  • 補不可能出現的數值 : 類別型欄位，但不適合⽤眾數時

歸一化和標準化

  • 歸一化:
    就是將訓練集中某一列數值特徵（假設是第i行）的值縮放到0和1之間[0 ~ 1]。
    最⼩最⼤化 (MinMax Scaler) :假定數值為均勻分佈，且資料不會有極端值、或已去極端值，適合本⽅式平衡特徵
    Rescaling (min-max normalization)：$ x’ = \frac{x - min(x)}{max(x) - min(x)} $
    Mean normalization：$ x’ = {\frac{x-average(x)}{max(x)-min(x)}} $

    from sklearn.preprocessing import MinMaxScaler
MinMaxScaler()
  • 標準化:
    就是將訓練集中某一列數值特徵（假設是第i行）的值縮放成均值為0，方差為1的狀態[-1 ~ 1]。
    假定數值為常態分佈，適合本⽅式平衡特徵
    Standardization：$ x’ = \frac{x - \bar{x}}{\sigma}$

  • 提高模型精度，這在涉及到一些距離計算的算法時效果顯著

連續型數值標準化

是否一定要做標準化 (有沒有做有差嗎)

標準化的意義：平衡數值特徵間的影響⼒

看使用的模型而定
• Regression model(如線性迴歸, 羅吉斯迴歸, ANN…等)：有差
• Tree-based model(如決策樹, 隨機森林, 梯度提升樹…等)：沒有太大關係

連續值標準化

  1. Z-transform: $ \frac{(x - mean(x))}{std(x)} $
  2. Range (0 ~ 1): $ \frac{x - min(x)}{max(x) - min(x)} $
  3. Range (-1 ~ 1): $ (\frac{x - min(x)}{max(x) - min(x)} - 0.5) * 2 $

標準化的好壞

Good

  • 某些演算法 (如 SVM, DL) 等，對權重敏感或對損失函數平滑程度有幫助者
  • 特徵間的量級差異甚大

Bad

  • 有些指標，如相關不適合在有標準化的空間進行
  • 量的單位在某些特徵上是有意義的

ECDF (experience empirical distribution function)

ECDF 最主要的目的是希望看到資料的累積分布，y 軸是介於 0-1 的數字 (代表累積比例)，x 軸則是資料的實際數值分布。
透過 ECDF，我們可以很快看出我們有興趣的資料數值約占多少比例，並觀察圖形差異
而CDF跟ECDF的差別就是，CDF 是從理論函數來的，ECDF 則是由觀察值所得到的，因此CDF看起來較為平滑，ECDF則較多曲折。

離群值(Outlier)及其處理理

何謂離群值(Outlier)

離群值是與正常數值偏離較遠的數值群，如果不處理則特徵縮放(標準化 / 最小最大化)就會出現很大的問題

檢查異異常值的⽅方法

  • 統計值：如平均數、標準差、中位數、分位數 (dataframe.describe)
  • 畫圖：如直方圖(hist)、盒圖(boxplot)、次數累積分布等(ECDF)

延伸資料：

Ways to Detect and Remove the Outliers
https://towardsdatascience.com/ways-to-detect-and-remove-the-outliers-404d16608dba

處理理異異常值

  • 取代補值：中位數、平均數等
  • 另建欄位
  • 整欄不用
  • 對數轉換
    對存在離群值的變量作對數轉換可以克服其離群值問題，且對數轉換並不影響各觀察值之間在此變量上的相對大小

    10 與 1000 相差百倍，但二者的對數值卻僅相差 3 倍，由此可知資料在經過對數轉換後變得更加集中
  • 縮尾
    將超出變量特定百分位範圍的數值替換為其特定百分位數值的方法。
  • 截尾
    將超出變量特定百分位範圍的數值予以刪除的方法。
  • 插值

處理完後的好處/壞處

好處是剩餘資料中模型較為單純且準確，壞處是有可能刪除掉重要資訊
